# 相談スキル

経営チームの専門家に相談し、多角的な視点から意見を集めるスキル。

## いつ使うか

- 「相談して」「みんなの意見聞いて」と言われたとき
- 事業判断に複数視点が欲しいとき
- 新機能・施策・コンテンツの評価をしたいとき
- 「これどう思う？」と経営的な相談をされたとき

## 専門家一覧

| Agent | 役割 | 得意領域 |
|-------|------|----------|
| `strategist` | 戦略担当 | 競合分析、長期視点、ポジショニング、リスク評価 |
| `marketer` | マーケ担当 | ターゲット分析、集客、KPI、トレンド |
| `uxdesigner` | UXデザイナー | ユーザー体験、UI、ブランド一貫性 |
| `writer` | ライター | コピー、メッセージ、トンマナ、読者目線 |
| `sales` | セールス担当 | 顧客視点、現場感覚、クロージング |

## 使い方

### 1. 議題を整理

ユーザーの相談内容を明確にする：
- 何について意見が欲しいのか
- 背景・制約条件は何か
- 判断基準は何か

### 2. 関連する専門家を選ぶ

**全員に聞く必要はない。** 議題に関連する専門家だけを選ぶ：

| 議題の例 | 呼ぶべき専門家 |
|---------|---------------|
| 新サービスの企画 | strategist, marketer, sales |
| LPのコピー | writer, uxdesigner, marketer |
| 価格設定 | strategist, sales, marketer |
| UI改善 | uxdesigner, writer |
| 競合対策 | strategist, marketer, sales |

### 3. Sub-agentとして呼び出す

各専門家を **Sub-agent** として起動し、同じ議題を投げる：

```
Task: 以下の議題について、あなたの専門的視点から意見をください。

## 議題
（ユーザーの相談内容）

## 背景
（コンテキスト）

## 判断したいこと
（何を決めたいか）
```

**注意:**
- 1人ずつ順番に呼ぶ（並列不可）
- 各専門家は独立して意見を出す（他の意見は見せない）

### 4. 意見を集約して報告

全員の意見が揃ったら、以下の形式で報告：

```markdown
## 経営チームの意見

### 議題
（元の相談内容を要約）

### 各専門家の意見

#### 🎯 Strategist（戦略）
（意見の要約）

#### 📊 Marketer（マーケ）
（意見の要約）

#### 🎨 UX Designer
（意見の要約）

#### ✍️ Writer
（意見の要約）

#### 🤝 Sales
（意見の要約）

### まとめ

**一致している点:**
- ...

**意見が分かれている点:**
- ...

**推奨アクション:**
1. ...
2. ...
```

## 注意事項

- **無限ループ防止:** 専門家同士の議論はさせない。各自が独立して意見を出す
- **コスト意識:** 全員に聞くとAPI呼び出しが5回。必要な専門家だけ選ぶ
- **最終判断は人間:** 専門家の意見はあくまで参考。決定権はユーザーにある
