# Claude Code スキル（Skills）ガイド

Claude Codeのスキル機能についてのリサーチ結果をまとめたドキュメント。

## スキルとは

スキル（Skills）は、Claudeが特定のタスクを繰り返し実行する方法を教えるための、**指示・スクリプト・リソースで構成されたフォルダ**。

Claudeに対して「このようなタスクが来たら、このように処理してほしい」という形式化されたテンプレートを提供する仕組み。

### 主な用途

- 会社のブランドガイドラインに従ったドキュメント作成
- 組織固有のワークフローでのデータ分析
- 個人タスクの自動化
- コードレビュー・セキュリティレビューの標準化

## スキルの構造

### ディレクトリ構成

```
.claude/skills/スキル名/
├── SKILL.md          # 必須：スキルの定義ファイル
├── scripts/          # オプション：実行スクリプト
└── references/       # オプション：参考資料やテンプレート
```

### 配置場所

| 種類 | パス | 対象 |
|------|------|------|
| 個人用スキル | `~/.claude/skills/` | 個人ユーザー |
| プロジェクト共有スキル | `.claude/skills/` | プロジェクトメンバー全員 |

### SKILL.md の構造

```markdown
---
name: スキル名
description: スキルの説明（15-20文字程度）
---

# スキルの詳細説明

このスキルがアクティブな場合にClaudeが従う指示をマークダウン形式で記述。

## 処理の流れ
- ステップ1
- ステップ2
- ステップ3

## 出力フォーマット
期待される出力形式を明確に定義。

## 注意点
重要な制約事項や注意事項を記述。
```

### 重要な要素

- **name**: スキルの表示名（`/name` コマンドとして表示される）
- **description**: スキルの簡潔な説明（マッチングに使用）
- **本文**: Claudeが実際に従う詳細な指示とプロセス

## スキルの実行方法

### 呼び出し方

```
/スキル名
```

例：
- `/review` - プルリクエストレビュー
- `/security-review` - セキュリティレビュー
- `/init` - CLAUDE.md初期化

### 実行フロー

1. ユーザーがスラッシュコマンドを入力
2. Claudeが該当スキルの description を参照
3. SKILL.md の指示を読み込み
4. 必要なスクリプトやリソースを実行
5. 結果をユーザーに返す

## 既存コマンドとの違い

### 従来のスラッシュコマンド

| 項目 | 内容 |
|------|------|
| 定義 | Claudeの基本機能（`/new`、`/edit`など） |
| 柔軟性 | 単純な操作指示、定義済みの動作 |
| カスタマイズ | 不可 |
| 用途 | 汎用的な基本操作 |

### スキル（Skills）

| 項目 | 内容 |
|------|------|
| 定義 | ユーザーが定義可能なカスタムコマンド |
| 柔軟性 | 複雑な業務プロセスの自動化が可能 |
| カスタマイズ | 完全にカスタマイズ可能 |
| 用途 | 組織・プロジェクト固有の要件に対応 |

### スキルの優位点

1. **テンプレート化された手順**
   - 繰り返しタスクを標準化
   - 人的ミスの削減

2. **リソース・参考資料の同梱**
   - scripts/ でスクリプト実行
   - references/ でテンプレート・ガイドライン参照

3. **チーム共有**
   - `.claude/skills/` でプロジェクト全体に共有
   - チーム全体での作業品質の統一

4. **組織固有のワークフロー対応**
   - ブランドガイドライン遵守の自動化
   - 組織のベストプラクティス定着

5. **スクリプト実行機能**
   - シェルスクリプト、Pythonスクリプトなどを組み込み可能
   - 外部ツールとの連携

6. **オープンスタンダード対応**
   - agentskills.io で公開標準化
   - OpenAI、GitHub Copilot など複数AIプラットフォームで対応

## 具体例

### セキュリティレビュースキル

```markdown
---
name: security-review
description: Complete a security review of the pending changes on the current branch
---

# Security Review Skill

## 処理手順
1. 現在のブランチの変更内容を取得
2. セキュリティ観点でコードを分析
3. 脆弱性や問題点を特定
4. レビュー結果をレポート形式で出力

## チェック項目
- SQLインジェクション
- XSS（クロスサイトスクリプティング）
- 認証・認可の問題
- 機密情報の漏洩リスク
- 依存パッケージの脆弱性
```

### PRレビュースキル

```markdown
---
name: review
description: Review a pull request
---

# PR Review Skill

## 処理手順
1. PRの変更内容を取得
2. コード品質をチェック
3. テストの網羅性を確認
4. レビューコメントを生成
```

## 作成時のベストプラクティス

1. **Description の最適化**: 簡潔で明確な説明（15-20文字）でマッチング精度を高める
2. **スクリプト分離**: scripts/ フォルダに実行スクリプトを分離し保守性向上
3. **参考資料の組織化**: references/ フォルダでテンプレートやガイド整理
4. **詳細な指示**: SKILL.md に詳細な指示と使用例を記述
5. **セキュリティ考慮**: 信頼できるソースからのスキル導入、インストール前の監査
6. **テスト**: 複数のユースケースで検証、エラーハンドリング確認
